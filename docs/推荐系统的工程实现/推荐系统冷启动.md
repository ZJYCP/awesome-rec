# 推荐系统冷启动

---

## 冷启动的概念

标的物和用户是不断变化的，对于新注册的用户，该推荐怎么样的标的物才能另其满意，这就是用户的冷启动。对于新入库的标的物，该怎么把新标的物分发出去给可能喜欢的用户，这就是标的物冷启动问题。对于新开发的产品，用户少，用户行为不多，不能很好的训练模型，如果让推荐变得越来越准确，这是系统的冷启动问题。

## 解决冷启动面临的挑战

对于新用户，我们基本上**不知道用户的真实兴趣**，主要因为：

1. App不强求用户注册时填写包含个人身份属性及兴趣偏好的信息
2. 用户是新注册的，没有过多的操作行为

对于新的标的物，**标的物信息不完善**，包含的信息不好处理，数据杂乱，或者新标的物产生的速度过快，短时间内来不及处理，或者处理成本很高。

对于新开发的产品，由于是从零开始发展用户，冷启动的问题更加突显，这时每个用户都是冷启动用户，面临的挑战更大。

## 解决冷启动的重要性

用户的不确定性需求是客观存在的，推荐是在目前信息爆炸的时代解决用户不确定需求的有效手段。推荐能有效提升用户的体验，增加新用户的留存。**对于互联网公司来说，用户是公司赖以生存的基础，是利润的核心来源。**互联网经济是建立在规模用户的基础智之上的，只有有足够的用户，产品才有变现的价值。

## 解决冷启动的方法和策略

### 用户冷启动

1. **提供完全非个性化的推荐**

   1. 利用先验数据做推荐

   推荐**新热标的物**。基于二八定律，20%的头部内容占到80%的流量，基于热门的推荐效果往往不错

   2. 给用户提供多样化的选择

   各种不同的类型的内容均推荐一个，类之间要有一定的区分度。

2. **利用用户在注册时提供的信息**

   1. 利用人口统计学数据

   年龄、性别、地域、学历、职业。鉴于目前的隐私安全问题，这种渠道还是比较困难的。

   2. 利用**社交关系**

   有些App在注册时可以导入社交关系，可以将用户的好友喜欢的标的物推荐给用户

   3. 利用用户填写的**兴趣点**

   在注册时用户选择兴趣点，借此进行推荐。需要注意的是，在填写兴趣点时，不能让用户填写过多内容，操作要简单。

3. **基于内容做推荐**

    当用户只有很少的行为时，算法往往无法给出较好的结果。基于内容的推荐，选择那些与用户已经点击过的内容相似的内容进行推荐。

4. **采用快速试探策略**

​		先随机或者按照完全非个性化推荐的方式给用户进行推荐，基于用户的点击反馈快速发现用户的兴趣点，从而在短时间内挖掘出用户的兴趣点，适合新闻和短视频之类的应用，内容短，用户可以快速切换。

5. **使用兴趣迁移策略**

​		通过用户在原有产品的兴趣特征迁移到新产品上，例如头条和抖音，淘宝和盒马鲜生

### 标的物冷启动

1. 利用标的物的metadata信息做推荐
   1. 利用标的物跟用户行为的相似性（用户看过的标的物特征的叠加），推荐给相似的用户
   2. 利用标的物跟标的物的相似性，推荐给消费过相似标的物的用户

2. 采用快速试探策略

   将新标的物曝光给一批随机用户，观察用户对标的物的反馈，找到对该标的物有正反馈的用户，后续将这标的物推荐给与这个用户相似的用户。

   该方法适合像淘宝这类的提供平台的电商平台以及像头条、抖音这种UGC 公司，他们需要维护第三方生态的繁荣。

   这种思路是一种**流量池**的思路，在不知道标的物是否受到喜欢的时候，先曝光给一批种子用户，看种子用户对标的物的反馈，如果反馈好，再曝光给更多的用户。

3. 采用基于关系传递的策略

​		举例：产品前期做长视频，后来有了短视频，利用用户的长视频观看历史，计算长视频和用户的相似度，对于新入库的短视频，计算与该短视频相似的长视频，再将该短视频推荐给与喜欢与它相似的长视频的用户。

### 系统冷启动

​	可以采用与用户冷启动相似的策略

### 新的视角审视冷启动

1. 产品初期，个性化推荐一定是必要的吗？

前期的个性化推荐没有那么重要，因为个性化推荐本身就是在有大量用户行为时效果才最好，前期可以做好搜索和导航，也可以做一个基于内容的标的物关联推荐，方便用户从内容关联到相似内容。

2. 跳出推荐的视角看用户冷启动

​	用户的需求：1.明确性需求，用户知道自己需要什么，这种情况产品可以通过搜索或者导航实现；另一类是不明确的需求，产品可以给出差异性较大的标的物让用户选择。

基于这个思路，可以给新用户设计一个独立的首次载入页面。

针对需求明确的情况，可以设计：热门模块 ，搜索模块 

针对需求不明确的情况，可以设计：筛选模块，主题模块



## 设计冷启动需要注意的问题

1. 逐步迭代让冷启动效果更好

​	通过AB测试选择一种效果好的方法，并不断优化

2. 量化冷启动用户的比例以及转换效果

​	进行AB测试，将用户的行为日志埋点，包含用户标识、算法标识、用户的具体行为等。可以通过日志分析每天的DAU中有多少用户采用了冷启动算法，各种冷启动算法的比例，以及转化效果

3. 采用级联推荐的策略

​	一般来说，协同过滤的效果好于基于内容的推荐，基于内容的推荐好于冷启动推荐。可以按这个顺序进行级联推荐。

4. 需要维护提供标的物的第三方利益

   采用流量池的思路

## 冷启动未来发展的趋势



1. 解决冷启动问题越来越迫切

创业朝精细化发展后，产品竞争越来越激烈，功能重叠会越多，替代品也会越来越多。要想让新用户留下来，需要优化推荐的冷启动方法。

2. 可以更加实时的了解用户的兴趣

对用户特征提取的速度将更快

3. 新技术下新的冷启动解决方案

语音、图像技术的发展、边缘计算能力的强大、数据安全法制建设的完善，可以快速通过视听特征构建用户画像

4. 资源共享的协同效应

不同APP产品可能达成同盟，利用共同的用户行为信息来协同优化用户体验。
